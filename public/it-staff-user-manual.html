Chapter 1

Getting Started — Login & Account Setup

How to access ClamFlow Cloud as IT Staff and what to do when login fails.

🔐 1.1   How to Log In

URL: https://clamflowcloud.vercel.app — open in any modern browser (Chrome recommended). Bookmark this address.

Procedure

  1. Open your browser and navigate to https://clamflowcloud.vercel.app.
  2. On the login screen, enter your Username (assigned by Admin, e.g. it_antony).
  3. Enter your Password (case-sensitive). Contact Admin if you have never received credentials.
  4. Click "Sign In". The system validates your credentials against the backend.
  5. On success, you are redirected to the IT Staff Dashboard automatically. Your name appears in the welcome banner.
  6. Your session is persisted in the browser. Closing the tab will NOT log you out — click your profile / logout button when done for the day.
⚠️ Shared computers: Always click Logout before leaving the workstation. Do not tick "Remember me" on shared devices.

First-Time Login

  1. Use the temporary credentials provided by Admin.
  2. If prompted to change your password, choose a strong password (minimum 8 characters, at least one number and one special character).
  3. Verify that your role shown in the dashboard header reads "IT Staff". If it shows a different role, contact Admin.

🔧 1.2   What To Do When Login Fails

⚠️ Troubleshooting — Login Issues
  • "Invalid credentials" Re-check username and password (caps-lock off). Try copy-pasting the password from the welcome e-mail. If still failing, ask Admin to reset your password in the User Management panel.
  • Page shows blank / loading spinner forever Clear browser cache (Ctrl+Shift+Del), hard-refresh (Ctrl+Shift+R), or try a private/incognito window. Check your internet connection.
  • "Something went wrong!" after login This is a dashboard crash. Open browser DevTools (F12 → Console) and screenshot the error. Share the screenshot with Admin. Usually caused by a backend deployment in progress — wait 2 minutes and retry.
  • Redirected back to login immediately Your session token may have expired. Log in again. If this happens every few minutes, check that your system clock is correct (token validation is time-sensitive).
  • Account locked / "User is not active" Contact Admin — your account's is_active flag may be set to false. Admin must reactivate it in the User Management panel.
  • Wrong dashboard shown (not IT Staff) Your role in the database does not match "IT Staff". Ask Admin to update your role via User Management → Edit User.
Chapter 2

Overview Tab 🖥️

Your at-a-glance system status, quick actions, and recent activity feed.

📊 2.1   How to Use the Overview

The Overview is the default tab when you log in. It gives you an instant snapshot of system status without navigating to other tabs.

System Health Banner

  • A colour-coded card shows overall backend status: Healthy, Warning, or Degraded.
  • Individual service badges (e.g. api, db, cache) show OK or DOWN.
  • The DB badge shows database connection state and response time in milliseconds.
  • If the banner is absent, the backend health check could not be reached — see §6.2.

Quick Actions Grid

  1. Four shortcut buttons are shown: View Devices, Hardware Config, RFID Handover, Audit Log.
  2. Click any button to jump directly to that tab — identical to clicking the tab at the top.

Recent Audit Events

  1. The last 5 system events are listed with: action name, user, role, time, and status badge.
  2. A success badge means the action completed normally. A failure badge means an action was attempted but rejected.
  3. To see the full log, click the Audit Log tab or use the Quick Action button.

🔧 2.2   What To Do When the Overview Fails

⚠️ Troubleshooting — Overview
  • Health banner missing entirely Backend health endpoint is unreachable. Click the System Health tab and hit Refresh. If still blank, backend may be starting up — wait 60 seconds. Escalate to Super Admin if the issue persists more than 5 minutes.
  • All services show DOWN Major backend outage. Do NOT attempt any write operations (handovers, onboarding). Document the time and notify Admin/Super Admin immediately.
  • "No audit events available" The audit log endpoint may be slow. Click the Audit Log tab and use its Refresh button. If still empty after refresh, this is a backend issue — note and escalate.
  • Quick Action buttons do nothing Hard-refresh the page (Ctrl+Shift+R). If JavaScript is blocked in your browser settings, enable it for this domain.
Chapter 3

Device Registry 📋

View, register, configure, and deregister all hardware devices in the facility.

🗂️ 3.1   How to Use Device Registry

Viewing All Devices

  1. Click the Device Registry tab in the navigation bar.
  2. The device list loads automatically. Each row shows: Device Tag, Type, Model, Serial Number, Status, and last assignment details.
  3. Device statuses:
    Available — Device is in stock, not assigned.
    Assigned — Device is currently with a staff member.
    Maintenance — Device is under repair and cannot be assigned.
  4. Use the search or filter controls (if present) to narrow by device type or status.

Registering a New Device

Supported types: RFID Reader, Camera, Biometric Scanner, Network Switch, Gateway, Scale, Printer, Tablet, PLC, Barcode Scanner, Temperature Sensor, pH Meter, Environmental Sensor.
  1. In the Device Registry tab, click "Register New Device" (or the + button).
  2. Fill in the registration form. Required fields are marked with *:
FieldDescriptionRequired
Device TagUnique identifier (e.g. DEV-001). Scan RFID or enter manually.✓
Device TypeSelect from the supported device type list.✓
ModelManufacturer model name (e.g. Zebra ZD410).✓
Serial NumberPhysical serial number printed on the device.✓
LocationCurrent physical location in the facility.
NotesAny special configuration notes or warranty info.
  1. Click "Register". A success message confirms the device is now in the system.
  2. The device will appear with status Available in the list.

Configuring an Existing Device

  1. Find the device in the list. Click its row or the "Configure" / edit icon on the right.
  2. Update the settings as needed (e.g. change location, update notes, toggle maintenance mode).
  3. Click "Save". A confirmation message appears.

Deregistering (Removing) a Device

⚠️ Deregistration is irreversible from the UI. Ensure the device is in Available status (not currently assigned) before proceeding.
  1. Confirm the device is not currently assigned to any staff member (check Handover tab → Active Assignments).
  2. Click the device row → select "Deregister" or the trash icon.
  3. Confirm the prompt. The device is removed from active inventory.

🔧 3.2   What To Do When Device Registry Fails

⚠️ Troubleshooting — Device Registry
  • Device list is empty on load Refresh the tab (navigate away and back). Check System Health — if the API service is DOWN, the list cannot load. Wait for backend recovery.
  • "Registration failed" / form submission error Ensure all required fields are filled. Check that the Device Tag is unique (duplicates are rejected). If the backend is returning 500 errors, escalate to Admin.
  • Device tag already exists error Search the existing list for that tag. If it's an old decommissioned device, ask Admin to permanently delete it from the database before re-using the tag.
  • Cannot deregister — assigned device Go to RFID Handover → Active Assignments, find and return the device first, then retry deregistration.
  • Changes saved but don't appear after refresh Browser cache issue — hard-refresh (Ctrl+Shift+R). If the data still reverts, this is a backend persistence issue — escalate with a screenshot.

🔧 4.5   What To Do When Hardware Config Fails

⚠️ Troubleshooting — Hardware Config
  • Settings don't load (404 in console) The backend configuration endpoint for this device type may not yet exist. Settings may only be saveable (first-time save creates the record). Try saving a default configuration to initialise it.
  • "Failed to save configuration" alert Check that all required fields are filled. Confirm your session hasn't expired (logout and login again). If the 500 error persists, escalate to Admin with the device type and field values you tried to save.
  • RFID reader not responding after config change 1) Disconnect and reconnect the reader's USB cable. 2) Verify the Connection Type matches the physical connection. 3) Check Device Manager (Windows) or lsusb (Linux) to confirm the OS sees the device. 4) Try the Generic USB HID model if the exact model isn't listed.
  • Face camera not detected Verify the camera is plugged in and has browser permission (chrome://settings/content/camera). Ensure no other application is using the camera. Reload the page.
  • Label printer prints garbage / incorrect format Verify the Model setting matches your exact printer. Check that the Label Format (ZPL/EPL) matches what the printer firmware expects. Confirm Paper Width matches loaded stock. Run a self-test print on the printer (hold feed button on power-up).
Chapter 5

RFID Device Handover 🔁

Issue RFID devices to station staff, process returns, and view assignment history.

📲 5.1   How to Issue a Device to Staff

Before you start: The device must be registered in Device Registry with status Available. The staff member receiving the device must have an active account in the system.
  1. Click the RFID Handover tab. The "New Handover" view loads by default.
  2. Step 1 — Identify the Device. You have two options:
    a) Scan: Click "Scan Device RFID Tag". The RFID reader must be connected and the device tag in range. The system will fill in the device details automatically.
    b) Manual: Type the Device Tag (e.g. DEV-001) into the "Enter Device Tag Manually" field. The device details appear below if found.
  3. Confirm the device card shows AVAILABLE. If it shows ASSIGNED, the device is already with someone — process a return first (§5.2).
  4. Step 2 — Select Staff Member. The right panel lists all active staff without a currently assigned device. Click the name of the staff member who will receive the device. Their details are highlighted.
  5. Review the Summary box at the bottom right — verify device tag, staff name, and station are correct.
  6. Click "✅ Complete Handover". A green success banner confirms: "Device DEV-XXX successfully handed over to [Name] at [Station]".
  7. The device status changes to Assigned and it moves to the Active Assignments tab.
⚠️ Physical handover: Ensure you physically hand the device to the staff member at the same time as completing the digital handover. Both records must match.

🔄 5.2   How to Process a Device Return

  1. In the RFID Handover tab, click "✅ Active Assignments" sub-tab.
  2. Find the device being returned (by device tag or staff name).
  3. Click the orange "🔄 Return Device" button on that row.
  4. Confirm the return in the prompt. The device status reverts to Available and it disappears from Active Assignments.
  5. Physically receive the device from the staff member and store it in the device cabinet.
  6. If the device is damaged, proceed to Device Registry → find the device → set status to Maintenance.

📋 5.3   Viewing Handover History

  1. In the RFID Handover tab, click "📋 History" sub-tab.
  2. Past handovers and returns are listed with: device tag → staff name, station, and return timestamp.
  3. If the history is empty, no devices have been returned yet (all currently active, or no handovers recorded).
Audit trail: All handover events are also logged in the Audit Log tab for compliance purposes.

🔧 5.4   What To Do When Handover Fails

⚠️ Troubleshooting — RFID Handover
  • Scan button doesn't read the tag 1) Ensure the RFID reader is physically connected and powered. 2) Check Hardware Config → RFID Reader settings are correct. 3) Hold the device tag within 5 cm of the reader antenna. 4) Try "Enter Device Tag Manually" as a fallback — scanning is optional.
  • "Device not found in system" The device tag scanned/entered does not exist in Device Registry. Register it first via the Device Registry tab, then retry the handover.
  • Device shows as ASSIGNED (can't hand over) Someone previously checked it out. Go to Active Assignments and process the return first, then do the new handover.
  • Staff member not appearing in the list The staff member may already have a device assigned, may be inactive, or may not be in the system. Check: 1) Active Assignments — if they already have a device. 2) Onboarding tab — if their account is pending Admin approval. 3) Ask Admin to verify the account is active.
  • "Handover failed" error on submit Backend handover endpoint is unavailable. Note the time and device/staff details. Complete the handover on paper (use the physical logbook) and record the details digitally once the system recovers. Escalate to Admin.
  • Handover recorded but Active Assignments doesn't update Navigate away and back to the RFID Handover tab to force a data reload. If still not showing, hard-refresh the page.

System Health Monitor ❤️

Real-time backend service status and database health diagnostics.

🩺 6.1   How to Read the System Health Panel

  1. Click the System Health tab. Health data loads automatically on page open.
  2. The Overall Status card shows one of three states:
StatusMeaningAction Required
Healthy ✅All systems operationalNone
Warning ⚠️One or more services degraded but system functionalMonitor closely; notify Admin if persists > 10 min
Degraded 🔴Critical service(s) offlineImmediate escalation to Admin/Super Admin required
  1. Services panel: Lists each backend service (api, cache, queue, etc.) as Operational or Down.
  2. Database panel: Shows connection status (connected / disconnected) and response time in milliseconds. Response time > 500ms is slow; > 2000ms is critical.
  3. Click "↻ Refresh" at any time to get the latest data.

✅ Healthy Response Time Guide

  • < 100 ms — Excellent
  • 100–300 ms — Good
  • 300–500 ms — Acceptable
  • 500–2000 ms — Slow — investigate network or DB load
  • > 2000 ms — Critical — escalate immediately

Uptime Display

The uptime value (e.g. "3 days 4 hours") shows how long the backend has been running without a restart. Unexpectedly low uptime (minutes) may indicate a recent crash or deployment.

🔧 6.2   Responding to Health Alerts

⚠️ Troubleshooting — System Health
  • "System health data unavailable" The health check endpoint cannot be reached. Check your internet connection first. Then wait 2–3 minutes — the backend may be restarting (Railway cold start). Click Refresh. If still unavailable after 5 minutes, escalate to Super Admin with current time.
  • One service shows DOWN Identify which service. If it is cache or queue, the system may still function — notify Admin but continue operations cautiously. If it is api or database, halt all write operations and escalate immediately.
  • Database response time > 2000 ms Heavy load or a slow query is blocking the database. Do not run large reports or batch imports. Notify Admin. If the response time doesn't improve within 10 minutes, escalate to Super Admin.
  • Services panel shows "No service data available" The backend's detailed health endpoint returned incomplete data. Click Refresh once more. This is usually transient (health endpoint returning before all services have initialized). If it persists after 3 refreshes, escalate to Admin.
  • Status shows Healthy but app features are broken The health check may be a "false positive" (backend is up but a specific feature endpoint is down). Check the Audit Log for recent errors. Open DevTools → Network tab and look for red (4xx/5xx) requests. Document and escalate with the specific endpoint that is failing.
Chapter 7

Audit Log 📜

Full trail of all system events — who did what, when, from which IP, and whether it succeeded.

🔍 7.1   How to Read the Audit Log

  1. Click the Audit Log tab. The most recent events load first.
  2. Each row contains:
    • Timestamp — Date and time of the event.
    • User — Full name of the person who performed the action.
    • Role — Their system role at the time.
    • Action — What was done (e.g. LOGIN, DEVICE_HANDOVER, WEIGHT_NOTE_CREATED).
    • IP Address — The source IP address.
    • Status — success or failure.
  3. Click "↻ Refresh" to pull the latest entries.
  4. For compliance investigations, note the Timestamp, User, Action, and IP Address — these are the key fields for a report.

Common Audit Actions to Monitor as IT Staff

Action CodeWhat it meansAlert if…
LOGIN_FAILEDSomeone failed to log inMultiple failures from same IP in short time (brute force attempt)
DEVICE_DEREGISTERA device was removed from registryUnexpected deregistration — verify with the person listed
CONFIG_CHANGEHardware config was updatedChange made by a non-IT Staff user
UNAUTHORIZED_ACCESSSomeone tried to access a restricted featureAny occurrence — escalate immediately
Tip: The audit log is read-only. You cannot delete or modify entries. This protects the integrity of the compliance record.

🔧 7.2   What To Do When the Audit Log Fails

⚠️ Troubleshooting — Audit Log
  • "No audit log entries available" Click the Refresh button. If still empty, check System Health — the audit endpoint may be down. This is an escalation item if it remains empty for more than 15 minutes during active use (the log should always have entries).
  • 404 error in browser console for /dashboard/audit/logs Backend routing issue. Do not attempt to fix from the frontend. Escalate to Admin/Super Admin with the exact error from browser DevTools → Network tab.
  • Log shows suspicious entries (unfamiliar users, odd IPs) Do not dismiss. Screenshot the suspicious rows. Report to Admin immediately with: Username, Timestamp, Action, IP Address. This may indicate an unauthorized access attempt.
  • Log only shows old entries (not refreshing) Click Refresh. If the timestamp of the latest entry doesn't update, the backend may be caching responses. Hard-refresh the browser page. Escalate if the issue persists — live audit data must be available for compliance.

Onboarding — Staff 👤

Submit new staff onboarding requests to Admin for approval and account activation.

👤 8.1   How to Submit a Staff Onboarding Request

⚠️ Approval flow: IT Staff can initiate onboarding but cannot activate accounts. All submissions are sent to Admin as Pending. Admin approves and activates the account, then completes Aadhar, Bank, and Face verification via the User Management panel.
  1. Click the Onboarding tab (last tab, 👤 icon). The "👤 Staff" sub-tab is shown by default.
  2. Read the amber approval notice banner to confirm you understand the flow.
  3. Fill in the New Staff Onboarding Request form:
FieldDescriptionRequired
Full NameStaff member's full legal name (as per ID)✓
DepartmentProduction / Quality Control / Security / IT / Other✓
RoleTheir job designation in the system (see role list below)✓
PhoneMobile number in +91 XXXXXXXXXX format
EmailWork or personal email for account credentials delivery
Start DateDate the person begins work✓
Initial StationWhich station / work area they are assigned to
Notes for AdminAny context useful for the Admin during verification (e.g. "joining as replacement for X", "urgent — starting tomorrow")

Available Roles for New Staff

  • Production Staff · Production Lead
  • QC Staff · QC Lead
  • Security Guard · Gate Staff
  • Maintenance Staff · Staff Lead
  • IT Staff
  1. Click "📤 Submit for Admin Approval". A green success banner confirms: "Onboarding request for [Name] submitted — pending Admin approval."
  2. The form resets automatically, ready for the next entry.
  3. Notify the staff member that their login credentials will be sent by Admin once the account is activated (typically within 1 business day).

What Happens Next (Approval Flow)

  1. Admin sees the new Pending request in User Management → Pending Onboarding.
  2. Admin reviews, verifies physical documents (Aadhar, bank details, face photo), and approves.
  3. Upon approval, a user account is created and login credentials are sent to the staff member.
  4. Admin completes the Aadhar and face verification steps via the "Complete Onboarding" modal in User Management.

🔧 8.2   What To Do When Staff Onboarding Fails

⚠️ Troubleshooting — Staff Onboarding
  • "Submission failed" error on submit Check all required fields (Full Name, Department, Role, Start Date) are filled. Verify you have an active internet connection. If the backend rejects the request, note the error message and escalate to Admin. As an offline fallback, record the details in the physical onboarding logbook.
  • "Saved offline" message appears This is expected if your device has no internet. The submission is queued locally and will sync automatically when the connection is restored. Do not submit again — this will create duplicates. Verify later that the entry appears in Admin's pending list.
  • Submission says success but Admin can't find it The online submission was accepted but the backend's pending list endpoint may be stale. Ask Admin to refresh/reload the User Management panel. If still not found after 10 minutes, re-submit and note it as a possible duplicate in the Notes field.
  • Wrong role accidentally selected and submitted You cannot edit a submitted request from the IT Staff dashboard. Contact Admin immediately and ask them to correct the role in the pending request before approving it.
  • "Access denied" / Onboarding tab not visible Your account's STAFF_ONBOARD permission may not be set. Ask Admin to verify your role is "IT Staff" (exactly, with capital I and capital S) in the user profile.
Chapter 9

Onboarding — Suppliers & Vendors 🚚

Register new boat owners and agents with Aadhar, bank details, and face recognition.

🚢 9.1   How to Register a Supplier / Vendor

The Suppliers / Vendors panel mirrors the Staff Lead's full onboarding workflow. All new registrations are submitted with Pending status and require Admin approval before the supplier can transact.

  1. In the Onboarding tab, click the "🚚 Suppliers / Vendors" sub-tab.
  2. The Supplier Onboarding panel loads with four sub-tabs: New Registration, Pending, Incomplete, Approved.

Creating a New Registration

  1. Click "➕ New Registration" tab.
  2. Select the supplier type: Boat Owner (primary supplier) or Agent (representative acting on behalf of a boat owner).
  3. Fill in the multi-step form. The form has these key steps:
    Step 1 — Basic Details: First name, last name, address, contact number, email, boat registration number (for boat owners), GST number.
  4. Step 2 — Aadhar Verification: Enter the 12-digit Aadhar number. Click "Confirm Aadhar". The system validates the format and marks it as attested (no OTP — verification is admin-attested).
  5. Step 3 — Bank Details: Bank name, account number, IFSC code, account holder name, and optional UPI ID.
  6. Step 4 — Face Registration: Click "Start Camera". Position the supplier's face in the video frame and click "Capture". Verify the captured image and accept it.
  7. Step 5 — Review & Submit: Confirm all details are correct. Click "Submit Registration". The record is saved as Pending.
  8. Admin will review and either approve or reject the registration. You can monitor status in the Pending tab.

Registering an Agent

  1. Select type Agent in the New Registration form.
  2. In addition to personal details, link the agent to an existing Boat Owner by selecting from the dropdown.
  3. Complete the Agent Declaration: agent name, phone, relationship to boat owner, authorized activities, and mark consent given.
  4. Submit — follows same Aadhar + Bank + Face flow.

Checking Pending & Incomplete Records

  1. Pending tab — submitted registrations awaiting Admin approval. Share the registration ID with Admin if they can't find it.
  2. Incomplete tab — registrations where one or more verification steps (Aadhar / Bank / Face) are missing. Return to these records to complete them before Admin can approve.

🔧 9.2   What To Do When Supplier Onboarding Fails

⚠️ Troubleshooting — Supplier Onboarding
  • Aadhar "invalid format" error The Aadhar number must be exactly 12 digits (no letters, no spaces, no dashes). Re-enter carefully. If the physical card has spaces, remove them before typing. Example: 1234 5678 9012 → enter as 123456789012.
  • Camera doesn't start for face registration 1) Allow camera permissions in the browser popup. 2) Check chrome://settings/content/camera and ensure this site is allowed. 3) Close other applications using the camera (e.g. Teams, Zoom). 4) Try a different browser. If no camera is available, note in the record and ask Admin to complete face registration at the admin workstation.
  • Face capture is blurry or rejected Ensure good lighting (face should be evenly lit, not backlit). Ask the person to remove glasses if the system cannot detect the face. Position them 40–60 cm from the camera. Try multiple captures and select the best.
  • Bank IFSC code validation error IFSC codes are exactly 11 characters (4 letters + 0 + 6 digits, e.g. SBIN0001234). Verify against the passbook or a bank IFSC lookup site. Ensure there are no spaces.
  • Duplicate registration warning The supplier may already exist. Check the Approved tab by name or boat registration number. If it's truly a new person (same name, different person), add a distinguishing note in the record (e.g. different contact number) and proceed.
  • Submission stuck on loading Wait up to 30 seconds. If still loading, check your internet connection. Do NOT click submit again (creates duplicates). Refresh the page — if the record appears in Pending, the submission succeeded. If not in Pending, try submitting again.
  • Record stays in Incomplete tab Click the record → complete the missing step(s) shown in red. Aadhar, Bank Details, and Face Registration are all required for Admin to be able to approve. Complete all three before informing Admin the record is ready for review.

🗂️ 10.2   Information to Collect Before Escalating

When reporting an issue to Admin or Super Admin, always provide:

  • ✅ Date and time the issue occurred
  • ✅ Your username and role
  • ✅ Which tab / feature was in use (e.g. "RFID Handover → New Handover")
  • ✅ Exact error message shown on screen (screenshot preferred)
  • ✅ Browser console errors: Press F12 → Console tab → screenshot any red errors
  • ✅ Network requests: Press F12 → Network tab → screenshot any red (4xx/5xx) requests
  • ✅ Steps to reproduce — exactly what you clicked before the error appeared
  • ✅ Browser and version (type chrome://version in Chrome)

⚡ 10.3   Quick Troubleshooting Checklist

Before escalating any issue, run through this checklist:

  1. Refresh the page — Ctrl+R or F5
  2. Hard-refresh (clears cache) — Ctrl+Shift+R
  3. Try a private/incognito window — rules out extensions/cache issues
  4. Check internet connection — open a different site (e.g. google.com)
  5. Check System Health tab — is the backend up?
  6. Log out and log back in — refreshes the session token
  7. Try a different browser — rules out browser-specific bugs
  8. If all above fail, escalate with screenshots

🔒 10.4   Security Reminders for IT Staff

These rules apply at all times. Violations must be reported to Admin.
  • 🔑 Never share your login credentials with anyone, including Admin.
  • 🚪 Always log out of ClamFlow before leaving your workstation unattended.
  • 📵 Do not attempt to access other roles' dashboards using test accounts.
  • 🛑 If you see "Access Denied" on a feature you should have access to, report it to Admin — do not attempt workarounds.
  • 🖥️ Do not install browser extensions that intercept network traffic on the ClamFlow workstation.
  • 📋 The Audit Log records ALL your actions. Every device handover, configuration change, and login is logged with your username and IP.
  • 🔐 If you suspect your account has been compromised (someone else logged in as you), change your password immediately and notify Admin.
